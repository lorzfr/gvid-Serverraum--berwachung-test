3. Szenario und Anforderungen

Ein Serverraum läuft rund um die Uhr. Zu hohe Temperaturen können Hardware beschädigen; gleichzeitig soll der Lüfter nicht unnötig Energie verbrauchen oder Lärm erzeugen.

Temperatur

Der TMP36 liefert eine analoge Spannung. Der Arduino wandelt diese in Grad Celsius um und glättet die Messwerte über einen gleitenden Mittelwert.

Lüfter

Der Lüfter wird über PWM angesteuert. Je höher die Temperatur zwischen 20 °C und 40 °C liegt, desto höher wird der PWM-Wert.

Personenerkennung

Der HC-SR04 misst den Abstand. Wird eine Person unter 50 cm erkannt, wird der Lüfter auf 0 % gesetzt.

4. Systemarchitektur und Hardware

Die Sensoren liefern Messwerte an den Arduino. Der Mikrocontroller berechnet daraus die Lüfterleistung und sendet Statusinformationen an das LCD.

Benötigte Komponenten

  • Arduino Atmega/Mega 2560
  • TMP36 Temperatursensor
  • 5V-Lüfter mit geeignetem PWM-/Treiberanschluss
  • HC-SR04 Ultraschall-Abstandssensor
  • 16x2- oder 20x4-LCD, idealerweise mit I2C-Adapter
  • Widerstände, Jumper-Kabel, Steckbrett
  • Empfohlen: Transistor/MOSFET und Freilaufdiode für reale Lüfterlasten

Sicherheits-Hinweis

Ein Arduino-I/O-Pin darf normalerweise keinen Lüfter direkt mit Betriebsstrom versorgen. Der Pin D9 sollte als Steuersignal verwendet werden. Für echte Lüfterlasten ist eine Treiberstufe mit Transistor oder MOSFET notwendig.

Der Ablauf ist deckungsgleich mit dem Code: Messwerte erfassen, Sensorwerte prüfen, Entscheidung treffen, Lüfter-PWM setzen und Anzeige aktualisieren.

Pseudocode

loop:
  wenn Temperatur-Intervall abgelaufen:
    TMP36 lesen
    Wert in Moving-Average-Puffer speichern

  wenn Ultraschall-Intervall abgelaufen:
    Entfernung messen
    personNearby = entfernung > 0 UND entfernung < 50

  wenn Sensorwert unrealistisch:
    Lüfter = 100 Prozent
  sonst wenn personNearby:
    Lüfter = 0 Prozent
  sonst wenn Temperatur-Hysterese aktiv:

  sonst:
    Lüfter = 0 Prozent

  wenn LCD-Intervall abgelaufen:
    Temperatur und Lüfter-Prozent anzeigen

6. Code-Dokumentation

Der Sketch sollte gut kommentiert, lesbar formatiert und in Funktionen aufgeteilt sein. Dadurch kann man in der Code-Abfrage jede Zeile erklären.

Wichtige Funktionen

  • readTemperatureC(): Wandelt den ADC-Wert des TMP36 in °C um.
  • smoothedTemperature(): Bildet einen Mittelwert über 10 Messungen.
  • readDistanceCm(): Misst Entfernung per HC-SR04.
  • calculatePWM(): Mappt Temperaturbereich auf PWM-Bereich.
  • updateFan(): Kombiniert Failsafe, Personenerkennung und Hysterese.
  • updateDisplay(): Aktualisiert LCD und Debug-Ausgabe.

Code-Qualitätsmerkmale

  • Aussagekräftige Konstanten wie TEMP_ON_C und PERSON_DIST_CM.
  • Keine magischen Zahlen in der Logik.
  • Nicht-blockierende Zeitsteuerung mit millis().
  • Structs bündeln Sensor- und Lüfterzustand.
  • Hilfreiche Kommentare erklären das Warum, nicht nur das Was.

Beispiel: Kernlogik der Lüftersteuerung

void updateFan(SensorData &s, FanState &f) {
  // Failsafe: Bei ungültigen Sensorwerten werden die Server geschützt.
  if (s.distanceCm < 0 || s.tempC < -20 || s.tempC > 80) {
    f.pwmValue = 255;
    f.percent = 100;
    f.enabled = true;
    analogWrite(PIN_FAN, f.pwmValue);
    return;
  }

  // Person zu nah: Lüfter wegen Lärm/Energieverbrauch abschalten.
  if (s.personNearby) {
    f.pwmValue = 0;
    f.percent = 0;
    f.enabled = false;
    analogWrite(PIN_FAN, 0);
    return;
  }

  // Hysterese: Einschalten ab 30 °C, Ausschalten erst unter 28 °C.
  if (!fanEnabledByTemp && s.tempC >= TEMP_ON_C) fanEnabledByTemp = true;
  if (fanEnabledByTemp && s.tempC <= TEMP_OFF_C) fanEnabledByTemp = false;

  f.pwmValue = fanEnabledByTemp ? calculatePWM(s.tempC) : 0;
  f.percent = map(f.pwmValue, 0, 255, 0, 100);
  f.enabled = f.pwmValue > 0;
  analogWrite(PIN_FAN, f.pwmValue);
}

Ein nachvollziehbares Testprotokoll zeigt, was getestet wurde, wie getestet wurde und welches Ergebnis erwartet wird.

Typische Fehler

  • LCD bleibt leer: Adresse 0x27 oder 0x3F prüfen.
  • HC-SR04 liefert 0: Verkabelung von TRIG/ECHO prüfen.
  • Lüfter startet nicht: Mindest-PWM erhöhen oder Treiberstufe prüfen.
  • TMP36-Werte falsch: Pinreihenfolge und 5V-Referenz kontrollieren.

Failsafe-Strategie

Bei ungültiger Temperatur oder fehlender Ultraschallmessung wird nicht einfach abgeschaltet. Stattdessen läuft der Lüfter sicherheitshalber mit 100 %, weil Server-Hardware vor Überhitzung geschützt werden muss.

9. Benutzeranleitung

Diese Kurz-Anleitung richtet sich an Personen, die den Prototyp zum ersten Mal starten oder vorführen.

1. Aufbau prüfen

Kontrolliere 5V, GND, A0, D3, D4, D9 sowie SDA/SCL. Der Lüfter sollte über eine passende Treiberstufe versorgt werden.

2. Sketch hochladen

Arduino IDE öffnen, Board „Arduino Mega or Mega 2560“ auswählen, richtigen Port wählen und Upload starten.

3. Anzeige lesen

Das LCD zeigt die aktuelle Temperatur und die Lüfterleistung in Prozent. Bei Personennähe erscheint eine Warnung bzw. die Lüfterleistung fällt auf 0 %.

4. Verhalten testen

Sensor erwärmen, Hand vor den Ultraschallsensor halten und Display-/Lüfterreaktion beobachten.
